# 禁用用户

> **禁用用户** 将**立即中断**目标用户的系统访问权限，同时保留其所有数据。适用于以下场景：
> - 员工离职或调岗期间的安全管控
> - 检测到异常登录行为
> - 临时性权限回收

## 操作步骤

### 方式一：用户列表页操作
1. 访问 `AngusGM → 组织人员 → 用户`
2. 在用户列表中找到目标用户
3. 点击该行右侧 **「···」** 操作菜单
4. 选择 **「禁用」**

### 方式二：用户详情页操作
1. 在用户列表中点击目标用户 **姓名**
2. 进入用户详情页面
3. 点击右上角 **「禁用」** 按钮

## 禁用后影响
| 功能权限       | 禁用状态     | 恢复后状态      |  
|----------------|----------|-----------------|  
| 系统登录       | ❌ 禁止     | ✅ 自动恢复      |  
| 数据访问       | ❌ 只读     | ✅ 完全恢复      |  
| 消息接收       | ❌ 正常     | ✅ 正常          |  
| 定时任务       | ⚠️ 按配置保留 | ⚠️ 按配置恢复    |  

## 管理操作
### 查看禁用账户
1. 在用户列表点击 **「状态筛选」**
2. 选择 **「禁用」** 选项

### 恢复账户
1. 找到目标禁用用户
2. 点击 **「启用」** 按钮
3. 权限即时恢复（需重新登录）
